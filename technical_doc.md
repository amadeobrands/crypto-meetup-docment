# 币聚

## 技术架构与方案选择

### 设计原则
* 前后端分离。分离粒度保持在 前端-后端即可，可以不需要业务中转层。
* 模块间松耦合，模块复用率要高。
* 要面向服务设计，采用微服务架构。
* 后端采用分布式架构，但要结合成本考虑，前期业务逻辑不复杂的时候，可以采用单虚拟机部署多个服务的方案。
